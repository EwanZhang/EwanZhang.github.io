我的博客是如何搭建的

Jun 30, 2016

博客部署在VPS（搬瓦工，系统是：CentOS 6 x86_64）上面，因为同时部署了图床，所以使用Nginx+PHP，如果不需要图床，使用Apache即可。GitHub只用作备份和备用线路。

一 . Nginx+PHP

如果不需要图床（qchan），可以跳过这一步，直接从“ 二 . Hexo ”开始。

安装前准备:
yum -y install gcc git openssl openssl-devel zlib zlib-devel

1.1 安装Nginx:

  1. 下载对应当前系统版本的nginx包
    # wget http://nginx.org/packages/centos/6/x86_64/RPMS/nginx-1.10.1-1.el6.ngx.x86_64.rpm

  2. 建立nginx的yum仓库
    # rpm -ivh nginx-1.10.1-1.el6.ngx.x86_64.rpm

  3. 启动nginx服务
    # systemctl start nginx

  4. 配置
    默认的配置文件在/etc/nginx (多站点可以在/etc/nginx/conf.d路径下配置*.conf)路径下，使用该配置已经可以正确地运行nginx；如需要自定义，修改其下的 nginx.conf 等文件即可。

  5. 测试
    在浏览器地址栏中输入部署nginx环境的机器的IP，如果一切正常，应该能看到如下字样的内容。

1.2 安装PHP

  1. 使用yum安装PHP
    # yum install php php-fpm php-bcmatch php-gd php-mbstring php-mcrypt php-mysql

  2. 配置
    PHP配置文件php.ini路径： /etc/php.ini

  3. 启动php-fpm
    service php-fpm start

二 . Hexo

2.1 建立GitHub Pages

GitHub Pages可以免费托管静态页面，自带主题、支持自制页面。
创建GitHub Pages需要有一个GitHub账号，然后创建一个仓库就可以了。但是仓库的命名格式必须为：yourusername.github.io（yourusername是你的GitHub用户名，下图中GitHub用户名为ewanzhang，所以仓库命名为ewanzhang.github.io），然后点击Crerte repository就可以了。

2.2 部署Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

2.2.1 安装Hexo

本步骤使用CentOS 6 x86_64，若为其他系统或有疑问，请参考：Hexo官方中文文档
